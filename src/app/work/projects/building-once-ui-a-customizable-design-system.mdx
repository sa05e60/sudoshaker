## Overview

Mirai’s artifact system keeps generation controlled and predictable. Planning is deterministic and cheap, validators normalize output, and exports never call the model.

## Major Product Work

- added centralized artifact constraints and deterministic validators for `slides`, `report`, `quiz`, and `flashcards`
- improved artifact quality without default multi-pass generation
- added artifact revision lineage, refine actions, version history, compare, and restore
- added deterministic exports:
  - slides -> `pptx`
  - reports -> `pdf`
  - flashcards -> `csv` and `tsv`
- added concept-check events and learning progression snapshots
- fed progression state into adaptive teaching policy generation
- strengthened recency weighting in behavioral and linguistic scoring
- added configurable artifact quotas, rate limiting, cooldowns, repeated-request protection, and revision caps

## Export Formats

- slides: `pptx`, `md`
- report: `pdf`, `md`
- flashcards: `csv`, `tsv`, `md`
- quiz: `md`

## Usage Control

- per-user daily and weekly artifact quotas
- short-window generation/refinement/export rate limits
- repeated-request abuse protection
- revision depth caps
- per-artifact cooldowns
- output-size and item-count limits by artifact type

## Outcome

Mirai now supports a bounded artifact workflow that is predictable, cheap, and easier to reason about in production.
